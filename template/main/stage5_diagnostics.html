Stage 5: The Request Forwarder

"You've reached the system diagnostics panel. It can fetch resources from any URL you provide. But what if you ask it to fetch from itself... or from hidden internal services?"

Hint: SSRF (Server-Side Request Forgery)

Recommended Fix — SSRF (Server-Side Request Forgery)

Implement a strict URL allow-list for any server-side fetch functionality. Block requests to private/internal IP ranges (127.x, 10.x, 172.16.x, 192.168.x) and cloud metadata endpoints. Isolate outbound network access and never expose raw fetch results to the user.
