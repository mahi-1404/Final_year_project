Stage 4: The Serialized State

"The system stores session state as serialized objects. It deserializes them without question. What if the object contains instructions instead of data?"

Hint: Insecure Deserialization (Java/Python/PHP)

Recommended Fix — Insecure Deserialization

Never deserialize untrusted data. Use signed, integrity-verified tokens (e.g., JWT with strong secrets) instead of raw serialized objects. Apply strict type allow-lists if deserialization is unavoidable, and run deserialization in a sandboxed context.
